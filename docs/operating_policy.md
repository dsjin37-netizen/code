# 운영 정책

본 문서는 주식 리서치 오케스트라 시스템의 전체 운영 원칙을 정의한다.

## 1. 시스템 목적

종목명을 입력받아 5개 전문 서브 에이전트의 분석을 메인 에이전트가 통합하여 한국어 투자 리서치 보고서를 생성한다. 본 시스템은 투자 참고용이며, 실주문·실매매·실계좌 연동은 별도 승인 전까지 금지한다.

## 2. 에이전트 구성

| 에이전트 | 역할 | 프롬프트 파일 |
|---|---|---|
| 메인 에이전트 | 전체 지휘, 통합, 최종 보고서 | prompts/main_agent.md |
| 재무 분석가 | 재무·회계품질·밸류에이션 | prompts/financial_analyst.md |
| 뉴스 감성 분석가 | 뉴스·공시·이벤트 감성 | prompts/news_sentiment_analyst.md |
| 업종 리서처 | 업종·경쟁사·정책 | prompts/sector_researcher.md |
| 공격적 투자 전략가 | 진입·청산 전략 | prompts/aggressive_strategy_agent.md |
| 리스크 감사관 | 반대논리·리스크 감사 | prompts/risk_auditor.md |

## 3. 실행 흐름

```text
사용자 입력
  └─ 종목명 / 시장 / 투자기간 / 보유 여부 / 투자성향 / 기준 가격
        ↓
메인 에이전트 (입력 정규화)
        ↓
1. 재무 분석가
2. 뉴스 감성 분석가
3. 업종 리서처
        ↓
4. 공격적 투자 전략가 (세 분석 통합)
        ↓
5. 리스크 감사관 (독립 검증)
        ↓
메인 에이전트 최종 검증 및 판정 조정
        ↓
최종 투자 판단 보고서
```

## 4. 투자 판단 원칙

```text
1. 출처 없는 수치는 사용하지 않는다.
2. 기준일 없는 데이터는 최종 판단 근거로 쓰지 않는다.
3. 재무·뉴스·업종 분석이 충돌하면 충돌 지점을 명시한다.
4. 손절 라인 없는 매수 의견은 금지한다.
5. 목표 수익률은 근거 없이 과장하지 않는다.
6. 실시간 데이터가 없으면 반드시 "데이터 기준일 확인 필요"로 표시한다.
7. 투자 판단은 확정 표현이 아니라 확률적 시나리오로 제시한다.
8. 리스크 감사관이 기각하면 최종 판정을 최소 Hold 이하로 낮춘다.
9. 유상증자, 감사의견, 상장폐지, 횡령, 대규모 소송은 최우선 위험 신호로 분류한다.
10. 실계좌 주문, 자동매매, 실매매 실행은 구현하지 않는다.
```

## 5. 입력 보완 규칙

필수 입력(종목명 또는 티커, 시장, 투자기간, 보유 여부)이 부족하면 임의 추정하지 않고 보완 질문을 생성한다.

```text
분석을 시작하려면 최소한 종목명 또는 티커, 시장, 투자기간, 보유 여부가 필요합니다.
현재 누락 항목: [누락 항목]
```

종목명만 입력된 경우에도 1차 리서치 보고서는 작성하되 다음을 명시한다.

```text
기준 가격, 보유 여부, 투자기간이 없어 진입·손절·포지션 크기는 예비안으로만 제시합니다.
```

## 6. 판정 체계

- Go: 데이터 충분, 분석 일관, 리스크 감사 통과
- Hold: 불완전·충돌·확인 필요
- No-Go: 치명적 리스크, 출처 누락, 감사 기각, 손절 설계 불가

세부 통합 매트릭스, 강제 하향 조건, 강제 보류 조건은 `prompts/main_agent.md`를 따른다.

## 7. 출력 언어

모든 사용자 대상 출력은 한국어로 작성한다. 단, 코드 변수명, 파일명, JSON 키, 함수명은 영어를 사용한다.
